3.1 - Creating React Functional Components And Rendering Elements

1 - What is Functional Component ?

A Component is a base building block of any react application.It is a JavaScript class or maybe a function which accepts inputs(properties-Props) and return React elements that shows how the User Interface need to be visible.

A Functional Component is a plain javascript function (Arrow and Regular Functions) that returns a JSX/HTML code. It behaves pretty much the same way as a Class Component but Class components have some additions. The Functional Component accepts props as an arguments and returns React Element. These functions may or may not receive data as parameters. In the Functional Components, the return value is the JSX code to render to the DOM tree.

# Here are the key reasons to start using functional components:

  • Functional components are easy to read and reason about
  • Functional components are easy to test
  • Functional components can potentially have a better performance
  • Functional components are easy to debug
  • Functional components are more reusable

2 - Creating a Simple Functional Component

We are going to create a simple functional component that help make the code easy by splitting the logic into reusable independent code. Always start your component names with a Capital Letter, because React treats components start with a lowercase letters as DOM Tags. For example <div> represent an HTML <div> tag, but <Welcome> represents a component that will be rendered in your DOM. Let's create our functional component.

# Creating a Functional Component that start with a Capital letter :-

➤ ./src/index.js


  // Importing the following libraries
  import React from "react";  <-- Now This is Optional to Import 
  import ReactDOM from "react-dom";
  
  function Welcome() {
    return <h2> Welcome to ReactJS </h2>;
  }

  // Render our component element into the DOM
  ReactDOM.render(<Welcome />, document.getElementById("root"));

# Components can be assigned to a variables :-

➤ ./src/index.js


  import ReactDOM from "react-dom"

  function Welcome() {
   return <h2> Welcome to ReactJS </h2>
  }
  
  const element = <Welcome />
  
  ReactDOM.render(element, document.getElementById("root"))

In the above example, we have created a function component called Welcome that return a h1 tag. The Component Welcome is used as an HTML tag. We can either add it into the render() method or assigned to element variable, then add the element variable into the render() method.

# Creating an App component that has other component inside it :-

➤ ./src/index.js


  import ReactDOM from "react-dom"

  const Welcome = () => <h2> Welcome to ReactJS </h2>
  
  function App() {
   return (
    <div>
     <Welcome />
    </div>
   )
  }
  
  ReactDOM.render(<App />, document.getElementById("root"))

3 - Exporting and Importing Components

We have learned that React application is a collection of interactive components, and these components can be re-used that may have been defined elsewhere. To do so we need to know two operations broadly known as Importing and Exporting. Instead od defining all of our components within the same where we render them, we can separate them into different files to make our application clean and simple. If you have a complex app and have to scroll through hundreds or thousands of lines of code, then the job of debugging or just understanding the app becomes that much harder.

A - Export Default And Export Only

When using export default on a function file. In order to import the export default function from that file, we can use only the address and use the keyword import before it, and also we can use only the export keyword, then we have import each name inside { } brackets separate each named parameter within the curly braces.

# Exporting and Importing export default function :-

➤ ./src/App.js


  export default function App() {}
  export default function App2() {} // Only one default export allowed per module.

➤ ./src/index.js


  import App from "./App.js"

# Using export default at the end :-

➤ ./src/App.js


  function App() { }
  export default App;

➤ ./src/index.js


  import App from "./App.js"

# Using export only :-

➤ ./src/App.js


  export function App() { }

➤ ./src/index.js


  import { App } from "./App.js"
  import App from "./App.js" // App does not contain a default export

# Using export with Arrow function :-

➤ ./src/App.js


  export const App = () => { }

➤ ./src/index.js


  import { App } from "./App.js"

# Using export default with Arrow function :-

➤ ./src/App.js


  export default const App = () => { } // Syntax Error
  const App = () => { }
  export default App

➤ ./src/index.js


  import App from "./App.js"

B - Multiple Exports

We can export several functions using the export keyword, then we can import all of them within the curly braces separate by comma. And also we can combine functions with export and default export keywords. let see the following examples.

# add Export beside each function and Importing several named parameters :-

➤ ./src/App.js


  export function App1() {}
  export function App2() {}

➤ ./src/index.js


  import { App1, App2 } from "./App.js"

# A combination of One default export function and other only exports functions :-

➤ ./src/App.js


  export default function App1() {}
  export function App2() {}
  export function App3() {}

➤ ./src/index.js


  import App1, { App2, App3 } from "./App.js"

# One default export and other exports at the end :-

➤ ./src/App.js


  export default function App1() {}
  function App2() {}
  function App3() {}

  export { App2, App3 }

➤ ./src/index.js


  import App1, { App2, App3 }  from "./App.js"

# Exporting multiple arrow functions using export keyword on each arrow function:-

➤ ./src/App.js


  export const App1 = () => { }
  export const App2 = () => { }
  export const App3 = () => { }

➤ ./src/index.js


  import { App1, App2, App3 } from "./App.js"

# Exporting multiple arrow functions at the end :-

➤ ./src/App.js


  const App1 = () => { }
  const App2 = () => { }
  const App3 = () => { }
  
  export { App1, App2, App3 }

➤ ./src/index.js


  import { App1, App2, App3 } from "./App.js"

4 - Composing Components - Re-use Components Many Times

Composing Components it's building components from other components. Components can refer to other components in their output. We can gather all other components into one main component. When a component contains another component it can be said that the parent component owns the child component.

# An Example of rendering components :-

➤ ./src/App.js


  function WelcomeMessage() {
    return <h1> Welcome </h1>;
  }

  function App() {
    return (
      <div>
        <WelcomeMessage />
        <WelcomeMessage />
        <WelcomeMessage />
      </div>
    )
  }

  export default App


  <div id="root">
    <div>
      <h1> Welcome </h1>
      <h1> Welcome </h1>
      <h1> Welcome </h1>
    </div>
  </div>

# An Example of rendering components into the App Component from different component files :-

➤ ./src/index.js


  import ReactDOM from "react-dom"
  import App from "./App"
  
  ReactDOM.render(<App />, document.getElementById("root"))

➤ ./src/App.js


  import { Header } from "./AllComponents/Header.js"
  import { Main } from "./AllComponents/Main.js"
  import { Footer } from "./AllComponents/Footer.js"

  function App() {
    return (
      <div>
        <Header />
        <Main />
        <Footer />
      </div>
    )
  }

  export default App

➤ ./src/AllComponents/Header.js


  export const Header = () => <h1> This The Header </h1>

➤ ./src/AllComponents/Main.js


  export const Main = () => <h1> This The Main </h1>

➤ ./src/AllComponents/Footer.js


  export const Footer = () => <h1> This The Footer </h1>  


  <div id="root">
    <div>
      <h1> This The Header </h1>
      <h1> This The Main </h1>
      <h1> This The Footer </h1>
    </div>
  </div>

In the example above, we have a single App component that contain other components the Header, Main and Footer components, our App component has these children, and all children has one parent component which the App Component. React components by design are meant to contain other React components.

5 - Nesting Components

In React, we can nest components inside within one another. The components that are nested inside parent components are called child components. This helps in creating more complex User Interfaces, and it's fast and easy to separate UI elements. In the following picture we have a parent component which has two children components, and one of the children components has other three child components.

# In the following parent/child relationship represent as a DOM tree :-

# An Example of nesting components :-

➤ ./src/index.js


  import ReactDOM from "react-dom"
  import App from "./App"
  
  ReactDOM.render(<App />, document.getElementById("root"))

➤ ./src/App.js


  import Header from "./Components/Header"
  import MainContent from "./Components/MainContent"
  
  function App() {
   return (
    <div>
     <Header />
     <MainContent />
    </div>
   )
  }

  export default App
