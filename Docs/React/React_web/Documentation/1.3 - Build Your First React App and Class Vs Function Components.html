2 - Understand package.json Scripts

If you go to the package.json file, you can see that the create-react-app has installed several react scripts for you. We have four main scripts "start", "build", "test" and "eject". We are going to view each script individually, in your project root directory you'll see a package.json file content which has all your react scripts as following.

# the content of package.json file, where you find your four main scripts :-


{
  ....
  "scripts": {
    "start": "react-scripts start",  
    "build": "react-scripts build",  
    "test": "react-scripts test",  
    "eject": "react-scripts eject"  
  }
  ....
}

Now that we know what a script is and where to find them, let’s dive into each one and explain what it does to a React app.

- start

The "start" script enables you to start the server on "development" NODE_ENV mode. When you run the following command react will open the app on http://localhost:3000 to view it in the browser.

# the start script enables you to start the server on development mode, using the following command line :-


> npm start

The command above will not only start the server, but it will also update live and display the latest change occurs every time you edit your react code. In addition, it will show lint errors on the terminal (console) when it fails to start the server in the form of meaningful error messages.

- build

After creating several files or components within separate folders. These separate files need to be merged or bundled to be precise in one single file. The "build" command script will build your react project on "production" NODE_ENV mode, it ensure that the finished project is bundled, minified and optimized. You must know that the "development " mode is not optimized. React create a new folder called "build" where all your final project files bundled, optimized and ready for production.

# the build script enables you to build your application on production mode :-


> npm build

The command above will Builds the app for production to the "build" folder. It correctly bundles React in production mode and optimizes the build for the best performance. The build is minified and the filenames include the hashes. Your app is ready to be deployed!

- test

The create-react-app uses Jest as a test runner. The "test" script enables you to launch the test runner in interactive watch mode. We will dive deep into testing react app later. But keep in mind that any file with .test.js or .spec.js extensions will be executed when the script is launched.

# the test script can be run on the terminal with the following commands :-


> npm test

- eject

The create-react-app comes with an excellent configuration that helps you build your react app with the best practices in mind to optimize it. When you run "eject" script it will remove the single build dependency from your project if you are not satisfied with the build tool and configuration choices that create-react-app provides for you. That means it will copy the configuration files and the transitive dependencies (Webpack, Babel, ESLint etc.) as dependencies in the package.json file. If you do that, you’ll have to ensure that the dependencies are installed before building your project.

# the eject script can be run on the terminal with the following commands :-


> npm eject

After running the command above, it won't be possible to run it again since all scripts will be available except the eject one. Use this command only if you need to. Otherwise, stick with the default configuration. It’s better, anyway.

4 - Understanding Functional Components vs. Class Components in React

Components let you develop the UI of the website independently and even we can use reusable components to implement a similar UI. In React there is two ways of writing a react component. One uses a function and the other uses a class. Recently functional components are becoming more and more popular. We will help you understand the difference between functional and class components.

A - Rendering JSX

The difference here is the syntax, a Functional Component is just a plain Javascript function or an arrow function that returns JSX. A Class Component is a JavaScript class that extends React.Component which has a render method.

# Define a functional component that returns JSX :-

➤ ./src/App.js


  import React from "react";

  // Plain Javascript Function 
  function FunctionalComponent() {
    return <h1> Hello, world </h1>;
   }

  // Arrow Function ES6 
  const FunctionalComponent = () => {
   return <h1> Hello, world </h1>;
  };

In other hand defining a Class Component, you have to make a class that extends React.Component. The JSX to render will be returned inside the render() method.

# Define a class component that returns JSX, with using Destructuring :-

➤ ./src/App.js


  import React, { Component } from "react";

  // Using Destructuring to extract Component class from react
  class ClassComponent extends Component {
   render() {
     return <h1> Hello, world </h1>;
   }
  }

# Define a class component that returns JSX, without using Destructuring :-

➤ ./src/App.js


  import React from "react";

  class ClassComponent extends React.Component {
   render() {
     return <h1> Hello, world </h1>;
   }
  }

B - Passing props

let’s see how they are written in both class and functional components. Inside a Functional Component, we are passing props as an argument of the function.But in Class Component you need to use "this" to refer to props. Let’s say we are passing props of the name "Adam" like below.

# Calling our component while passing props :-

➤ ./src/App.js


  <FunctionalComponent name="Adam" />
  <ClassComponent  name="Adam" />

# Passing props with functional component and using Destructuring :-

➤ ./src/App.js


  const FunctionalComponent = ({ name }) => {
    return <h1> Hello, {name} </h1>;
  };

  const FunctionalComponent = (props) => {
    return <h1> Hello, {props.name} </h1>;
   };

# Passing props with class component and using "this" to refer to props :-

➤ ./src/App.js


  class ClassComponent extends React.Component {
    render() {
      const { name } = this.props;
      return <h1> Hello, { name } </h1>;
   }
  }

C - Handling state in Functional Component And Class component

To use state variables in a Functional Component, we need to use useState Hook, which takes an argument of initial state. in the following example a useState hook takes an argument of initial state. We start with 0 clicks so the initial state of count will be 0.

# Handling state by using state hooks in functional components :-

➤ ./src/App.js


  const FunctionalComponent = () => {
    const [count, setCount] = React.useState(0);
   
    return (
      <div>
        <p>count: {count}</p>
        <button onClick={ () => setCount(count + 1)}>Click</button>
      </div>
    );
   };

The initial state can be null, string and even object. The useState hook return the current state using the count variable, and a function setCount() that updates it, we are destructuring the array two elements of the array consider them as a state and its setter.

Handling state in a Class Component is a bit different, we define a constructor first. Inside the constructor you will make a state object with a state key and initial value. And inside JSX, we use this.state.count to access the value of the state key we defined in the constructor to display the count. Setter is pretty much the same, just different syntax.

# Handling state by using setState method in class components :-

➤ ./src/App.js


  class ClassComponent extends React.Component {

    constructor(props) {
      super(props);
      this.state = {
        count: 0
      };
    }
   
    render() {
      return (
        <div>
          <p>count: {this.state.count} times</p>
          <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          <button onClick={() => this.setState((state) => { return { count: state.count + 1 } } })}>
            Click
          </button>
        </div>
      );
    }
   }

D - Lifecycle Methods

Lifecycles play an important role in the timing of rendering. For those of you who are migrating from class components to functional components, you must be wondering what could replace lifecycle methods such as componentDidMount() in a Class Component And yes, there is a hook that works perfectly for the purpose, let’s check it out!

- componentDidMount() and useEffect(() => { }, []) Hook

The lifecycle method componentDidMount() is called right after the first render completes. There used to be a componentWillMount that happens before the first render, but it is considered legacy and not recommended to use in newer versions of React.

# Invoke componentDidMount using the useEffect hook :-

➤ ./src/App.js


  const FunctionalComponent = () => {
    React.useEffect(() => {
      console.log("Hello");
    }, []);
    return <h1> Hello, World </h1>;
  };

Replacing componentDidMount(), We use the useEffect hook with the second argument of []. The second argument of the useState hook is normally an array of a state(s) that changes, and useEffect will be only called on these selected changes. But when it’s an empty array like this example, it will be called once on mounting. This is a perfect replacement for a componentDidMount().

# Using componentDidMount that will be invoked after the first render :-

➤ ./src/App.js


  class ClassComponent extends React.Component {
    componentDidMount() {
      console.log("Hello");
    }
   
    render() {
      return <h1>Hello, World</h1>;
    }
   }

- componentWillUnmount() and useEffect(() => { return () => { ... } }, [] ) Hook

Now using componentWillUnmount() syntax is a bit different in a Functional Component, we can use a useState hook for unmounting. But be careful, the syntax is a bit different. What you need to do is return a function that runs on unmounting inside the useEffect function. This is usefull especially when you have to clean up the subscriptions such as a clearInterval function, otherwise it can cause a severe memory leak on a bigger project. One advantage of using useEffect is that we can write functions for both mounting and unmounting in the same place.

# Using useEffect to unmount components :-

➤ ./src/App.js


  const FunctionalComponent = () => {
    React.useEffect(() => {
      return () => {
        console.log("Unmount Successfully");
      };
    }, []);
    return <h1> Bye, World </h1>;
    };

# Using componentWillUnmount method to unmount components :-

➤ ./src/App.js


  class ClassComponent extends React.Component {
    
    componentWillUnmount() {
      console.log("Unmount Successfully");
    }
   
    render() {
      return <h1> Bye, World </h1>;
    }
   }

5 - When to use the Class Component and the Functional Component ?

To answer this question we need to have a decent understanding of the state of the component. In Class Component, the render method will be called, whenever the state of the components changes. On the other hand, the Functional components render the UI based on the props.

Whenever we have the use case with the State of the component and rendering the UI based on the Component State, use the Class Component. It should be preferred whenever we have the requirement with the state of the component.

In React 16.8 we can use state in Functional Components also, so there is no need to use the class components. In version 16.8 Hooks were introduced and with the use of useState hook we can use the state in Functional Components also. It is advised that you use Functional Components rather than Class Components by Facebook itself.
