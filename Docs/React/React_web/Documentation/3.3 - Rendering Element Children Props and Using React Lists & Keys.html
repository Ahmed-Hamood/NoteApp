1 - Rendering Element Children Props

Props children used to display whatever you include between the opening and closing tags when invoking a component. the props.children is a special prop, automatically passed to every component, that can be used to render the content included between the opening and closing tags when invoking a component.

All Child elements must be placed between the two tag component like this <MyComponent> child Elements </MyComponent>. When a component has no children, you can invoke it with either <MyComponent></MyComponent> or <MyComponent/>, but the prefer is using self closing tag to make code shorter and easier to read.

# Example of Using children props :-

➤ ./src/index.js 


  import React from "react"
  import ReactDOM from "react-dom"

   return (

     <img src={props.src} />

2 - Rendering List of Elements using the map() Method

Rendering a list of element is really simple, we just use the build-in javascript map() method inside the JSX using curly braces to render list of items by iterating over an array of numbers, strings or objects, so it iterate over every element of that array, then after finish iterating it creates a new array with transformed values. It doesn’t change the parent or main array.

# An example of assigning the new array returned by map() to the variable result and log it:

➤ An Example 


  const numbers = [1, 2, 3, 4, 5];
  const result = numbers.map((number) => number * 2);
  console.log(result); // [2, 4, 6, 8, 10]

# An Example of Iterate thru an Array with react:

➤ ./src/index.js 


  const numbers = [1, 2, 3, 4, 5];

    <li> {number} </li>
  );

    <ul> {listItems} </ul>,
    document.getElementById("root")

    3 - Rendering Lists with Keys

    When creating a list of elements each element need a unique identifier like a Key property and it's value should be a string. A Key is a special string attribute you need to include when creating a lists of elements. Keys give the elements a stable identity and each list should be unique mean each key can't have the same value. Keys is kind of the id for the element that help React identify which items have changed, added, or removed help in efficient update of the user interface.

    # Using Key props to identify each list element :-

    ➤ An Example

    
  const numbers = [1, 2, 3, 4, 5];
  const listItems = numbers.map((number) =>
    <li key={number.toString()}>
      {number}
    </li>
  );

    ➤ An Example

    
  const list = ["a", "b", "c"];
 
  const SimpleList = () => (
    <ul>
      {list.map(item => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );

    # Let’s assign a key to our list items inside numbers.map() and fix the missing key warning issue :-

    ➤ ./src/index.js 

    
  import React from "react"
  import ReactDOM from "react-dom"
  
  function NumberList(props) {
   const numbers = props.numbers
   const listItems = numbers.map(number => <li key={number.toString()}> {number} </li>)
   return <ul> {listItems} </ul>
  }
  
  const numbers = [1, 2, 3, 4, 5]
  ReactDOM.render(<NumberList numbers={numbers} />, document.getElementById("root"))

    # If you don't have stable IDs for rendered items, you may use the item index as a key as a last resort as following.

    ➤ ./src/index.js

    
  import React from "react"

  const tasks = [
   { text: "Buy flowers", status: "todo" },
   { text: "Go to the dentist", status: "done" },
  ]

  const TaskItems = tasks.map((todo, index) => (
   // Only do this if items have no stable IDs
   <li key={index}> {todo.text} </li>
  ))
  
  ReactDOM.render(<div> {TaskItems} </div>, document.getElementById("root"))

    4 - Conditional rendering

    You may want to render conditionally render components base on the value of properties. like only you want to render something when the user is logged in. Conditional rendering in React works the same way conditions work in JavaScript. Use JavaScript operators like if or the conditional operator to create elements representing the current state, and let React update the UI to match them.

    # Use if-else Condition :-

    ➤ An Example

    
  if(props.isLoggin){
    return <div> Welcome Adam </div>
  } else{
    return <div> Welcome Adam </div>
  }

    # Or Use Ternary Condition :-

    ➤ An Example

    
  return this.isLoggin ? <div> Welcome Adam </div> : <div> Welcome Adam </div>

    # Store component in variable :-

    ➤ An Example

    
  let message;

  if(props.isLoggin){
    message = <div> Welcome Adam </div>
  } else{
    message = <div> Welcome Adam </div>
  }

  return message;

    # An Example of Conditional rendering :-

    ➤ ./src/index.js

    
  import React from "react"
  import ReactDOM from "react-dom"
  
  function UserGreeting(props) {
   return <h1> Welcome back! </h1>
  }
  
  function GuestGreeting(props) {
   return <h1> Please sign up. </h1>
  }
  
  function Greeting(props) {
   const isLoggedIn = props.isLoggedIn
   if (isLoggedIn) {
    return <UserGreeting />
   }
   return <GuestGreeting />
  }
  
  ReactDOM.render(<Greeting isLoggedIn={false} />, document.getElementById("root"))

    
  <div id="root">
    <h1>Please sign up. </h1>
  </div>

    # An Example of Using Conditional rendering wit ternary Operator :-

    ➤ ./src/index.js

    
  import React from "react"
  import ReactDOM from "react-dom"
  
  function UserGreeting(props) {
   return <h1> Welcome back! </h1>
  }
  
  function GuestGreeting(props) {
   return <h1> Please sign up. </h1>
  }
  
  function Greeting(props) {
   const isLoggedIn = props.isLoggedIn
   return <div> {isLoggedIn === true ? <UserGreeting /> : <GuestGreeting />} </div>
  }
  
  ReactDOM.render(<Greeting isLoggedIn={false} />, document.getElementById("root"))  

    
  <div id="root">
    <h1>Please sign up. </h1>
  </div>
