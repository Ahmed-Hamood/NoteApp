1.1 - What's ReactJS, Installation, Folder Structure & Start React App

1 - What is React ?

React (sometimes called React.js or ReactJS) is a JavaScript library for building fast, interactive, efficient and flexible user interfaces. It's an open source component base front-end JavaScript library responsible only for the view layer of the application. It's maintained by Facebook and a community of individual developers and companies. It is widely used as a base in building single-page websites and mobile applications. It is very easy to use, and it allows users to create reusable UI components.

React designs a simple views for each state in your application, and React will efficiently update and render just the right component when your data changes. Think of component as a simple function that you can call with some input and they render some output. And as you can reuse functions, so also you can reuse components, merge them and thereby creating a complex user interface.

# Here is an example of cutting a page into components, each section highlighted above is considered a component:

We can say that React is a way to build user interfaces, it's only concerned with what you see on the front-end. React makes user interfaces very easy to build by cutting each page into pieces. We call these pieces components (learn later).

React is a lightweight view library that only supports the UI components. Therefore, it requires integration with other technologies and tools (such as Flux and Redux) in order to get a full-fledged architecture.

2 - Features of React, and it's Advantages

There are unique features are available on React like JSX, Virtual DOM, Component, Performance and One-way Data Binding. Because of These features that's why React is widely used and popular. Let's take a look at these features.

Use JSX (Javascript XML), it's an extension of JavaScript it's faster than normal javascript as it performs optimizations while translating to regular JavaScript. It makes it easier for us to create templates. Though it is not mandatory to use JSX in react, it is one of the good features and easy to use.

Virtual DOM exists because it's a lightweight copy of the actual DOM. a virtual DOM is the representation of an original DOM. React creates a virtual dom in-memory data-structure cache. Only the final changes of DOM has later updated in the browsers DOM. Manipulating DOM is slow, but manipulating Virtual DOM is fast as nothing gets drawn on the screen.

One-way Data Binding This feature gives you better control over your application. React uses a unidirectional data flow. Meaning, the developers cannot edit any component directly. They have to utilize the callback function in order to make changes in the components. This process is called one-way data binding.

Components are like pure JavaScript functions that help make the code easy by splitting the logic into reusable independent code. A Component is one of the core building blocks of React. It'll make the task of building UIs much easier. You can see a UI broken down into multiple individual pieces called components and work on them independently and merge them all in a parent component which will be your final UI.

Performance React.js use JSX, which is faster compared to normal JavaScript and HTML. Virtual DOM is a less time taking procedure to update webpages content. all these features makes react perform fast and efficient.

# Advantages of React :-

  • React uses virtual dom that makes use of in-memory data-structure cache, and only the final changes are updated in browsers dom. This makes the app faster.
  • You can create components of your choice by using the react component feature. The components can be reused and also helpful in code maintenance.
  • React is an open-source javascript library, so it is easy to start with.
  • React has become very popular in a short span and maintained by Facebook and Instagram. It is used by many famous companies like Apple, Netflix, etc.
  • Facebook maintains ReactJS, the library, so it is well maintained and kept updated.
  • React can be used to develop rich UI for both desktop and mobile apps.
  • Easy to debug and test as most of the coding is done in Javascript rather than on Html.

3 - Using ReactJS from CDN

To start working with react, we need to first install reactjs. You can easily get started to use reactjs by using the CDN javascript files, as shown below. Go to the official site of reactjs to get the CDN links, Click here

# The following CDN versions are meant only for development, and are not suitable for production:-

  <script crossorigin src="https://unpkg.com/react@17/umd/react.development.js"> </script>
  <script crossorigin src="https://unpkg.com/react-dom@17/umd/react-dom.development.js"> </script>

# The following CDN versions are minified and optimized for production used :-

  <script crossorigin src= "https://unpkg.com/react@17/umd/react.production.min.js"> </script>
  <script crossorigin src= "https://unpkg.com/react-dom@17/umd/react-dom.production.min.js"> </script>

To load a specific version of react and react-dom, replace 17 with the version number. if you are planning to use the CDN files, make sure to keep the cross-origin attribute, to avoid cross-domain issues. Reactjs code cannot be executed directly in the browser and needs to be transpired using Babel to javascript before executing in the browser.

Start by including three scripts, the first two let us write React code in our JavaScripts, and the third, Babel, allows us to write JSX syntax and ES6 in older browsers.

# TAlso include Babel so it allow us to write JSX syntax :-

  <script src= "https://unpkg.com/babel-standalone@6.15.0/babel.min.js"> </script>

# Here is the working ReactJS example using cdn scripts files and babel-js script :-

  <!DOCTYPE html>
  <html>
  <head>
    <meta charset="UTF-8" />
    <title>Add React in One Minute</title>
  
    <!-- Load React. -->
    <!-- Note: when deploying, replace "development.js" with "production.min.js". -->
    <script crossorigin src="https://unpkg.com/react@17/umd/react.development.js"></script>
    <script crossorigin src="https://unpkg.com/react-dom@17/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/babel-standalone@6.15.0/babel.min.js"></script>
    
  </head>
  <body>
  
    <!-- We will put our React component inside this div. -->
    <div id="app"> </div>
  
    <!-- Load our React component. -->
    <script type="text/babel">
      ReactDOM.render(
        <h1>Hello, from React Tutorials!</h1>,
        document.getElementById('app')
      );
    </script>

  </body>
  </html>

5 - Install React Using npx create-react-app [project_name]

You can create a React application by installing the create-react-app tool-chain command. Facebook has created Create React App environment that will create a live development server and automatically compile React and all other things. You can install each react package like in our previous example, or use create-react-app to get the latest packages with other features.

# The create-react-app Syntax to install you react project :-

  npx create-react-app <Folder_Name>

# The following Steps on how to install react application in order to start your project :-

  • Install Node.js and NPM globally on your machine, go to node js website and download it Click here
  • We create our project folder let’s call it "my-react-app"
  • Open our "my-react-app" folder using vs-code"
  • Run "npm init --yes" to initialize your package.json
  • Then Use create-react-app to install react thru CLI Terminal"

To create a new React project, we can use the tool npx, provided you have an npm version of at least 5.2. npx gives us the ability to use the create-react-app package without having to first install it on our computer, which is very convenient. Using npx also ensures that we are using latest version of Create React App to create our project:

You can check what npm version you have by running in your terminal "npm -v" command.

# Run the following command to start creating your react application latest version and features :-

  npx create-react-app my-react-app
  // Create and install react application within the current folder using '.' dot
  npx create-react-app .
  npm start

Once we run this command, a folder named "my-react-app" will be created where we specified on our computer and all of the packages it requires will be automatically installed. The create-react-app will set up everything you need to run a React application. And make sure you should not use uppercase letters for your application name.

If you’ve previously installed create-react-app globally via "npm install -g create-react-app", we recommend you to uninstall the package using "npm uninstall -g create-react-app" to ensure that npx always uses the latest version

Create React App doesn’t handle backend logic or databases; it just creates a frontend build pipeline, so you can use it with any backend you want. Under the hood, it uses Babel and webpack, but you don’t need to know anything about them.

# To create a React app that uses TypeScript, we can use the Create React App TypeScript template :-

  npx create-react-app my-react-app --template typescript

6 - Reviewing the Project Folder Structure

Once our react project files have been created and our dependencies have been installed. Let’s walk through the directory structure that React CLI created for us. create react app will initial react framework/library files our project structure should look like this. Create react app does use webpack which is a module bundler it uses babel behind the scene

# The project folder files and directories :-

A - node_modules

This Folder includes all of the dependency-related code that Create React App has installed.this will have all the library dependency of package.json. You will never need to go into this folder.

B - package.json

it manages our app dependencies and what is included in our node_modules folder for our project, plus the scripts we need to run our app. Records of our project dependencies and configures our project

# the content of package.json file :-

  {
    "name": "ReactApp",
    "version": "0.1.0",
    "private": true,

C - package-lock.json

Records the exact version of packages that we install. It describes the exact tree that was generated, such that subsequent installs are able to generate identical trees, regardless of intermediate dependency updates.

D - .gitignore

A file that is used to exclude files and folders from being tracked by Git. We don't want to include large folders such as the node_modules folder

E - public

A folder that we can use to store our static assets, such as images, svgs, and fonts for our React app. In this directory our important file is index.html As you know React is a single page application so it is the only one Html or you can say a single file that gets rendered when we run the application. There is a "serviceWorker.js" file is a service worker for PWAs — Progressive Web Apps.

# In the public folder we have an index.html file :-

  <!DOCTYPE html>
  <html lang="en">
    <head>
      <meta charset="utf-8" />
      <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />

This <div id="root"></div> will be replaced with the root component. has the div#root which is basically where our entire react application is going to output. It’s going to output the main app component or root component code let’s say any component we create like nav bar a user list etc.. will get embedded on the main component or div#root

F - src

A folder that contains our source code. It is where all of our React-related code will live and is what we will primarily work in to build our app. We have the "App.js" file that contain all our React code. It has our component, which will get render when we run our application.

The index.js file has a method the ReactDOM.render() that renders the index.html root div. It will replace this root div with the app component. This is how your first component will work. as following

# A render method that render our html root id :-

  import React from "react"; // import react library
  import ReactDOM from "react-dom"; // import react Dom To render browser
  import "./index.css"; // import index style 
  import App from "./App"; // import App Component 
  import * as serviceWorker from "./serviceWorker"; // Progressive Web Apps 
  
  ReactDOM.render(<App />, document.getElementById('root') );
  
  serviceWorker.unregister();
