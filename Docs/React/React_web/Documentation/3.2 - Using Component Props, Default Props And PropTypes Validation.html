3.2 - Using Component Props, Default Props And PropTypes Validation

1 - What are Component Props with Examples ?

Props is short for properties and they are used to pass data between React components and they are optional and it makes the component dynamic. Props are arguments passed into React component, it's similar to function argument in Javascript and Attribute in HTML. Props are properties as a global object or variables which can be used inside a component. To send props into our Component we call our component and pass Attributes or argument into that component.

# The props syntax that we pass data to the component props :-

React's data flow between components is uni-directional (from parent to child only). Props Attributes string value wrap around double quotes "", and when assigning number or variable we wrap them a round curly braces {}.

# Here is a simple example of passing props to a Function Component :-

➤ ./src/index.js 


  import ReactDOM from "react-dom"
 
  const App = () => {
    const UserId = "4522584"

    return(
      <div>
        <h1> Welcome To ReactJS </h1>
        <UserInfo username="Adam Sam" age={45} UserId={UserId} />
      </div>
    )
  }

  function UserInfo(props) {
    console.dir(props);  ==> { age: 45, username: "Adam Sam", UserId: "4522584" } 
    return(
      <div className="InfoContent" >
        <p> Username = { props.username } </p>
        <p> Age = { props.age } </p>
        <p> User ID = { props.UserId } </p>
      </div>

  ReactDOM.render(<App />, document.getElementById("root"))


  <div id="root">
    <div>
      <h1> Welcome To ReactJS </h1>
      <div class="InfoContent">
        <p> Username = Adam Sam </p>

        <p> User ID = 4522584 </p>
      </div>
    </div>

# How Props works :-

We have added two Attributes username, age and UserId to UserInfo Component. And then we access all Attributes thru props inside our UserInfo Component. As you see props output is an object that has all the Attributes that we passed within the UserInfo Component.

4 - Pass Event Handler Function into Props

We know that we can pass string, Array or Object data to other components thru props. Also we can pass a callback function to other components from our parent Component thru props. see the following Examples with functional Components pass callback function into the child component thru props, then use the callback function to pass data back to the Parent component

# An Example of Pass a function as props :-

➤ ./src/index.js 


  import React from "react"
  import ReactDOM from "react-dom"
  
  function App() {
  const GetUserInput = (name, age) => {
    console.log(name, age) 
  }
  
   return <ChildComponent PassGetUserInput={GetUserInput} />
  }
  
  function ChildComponent(props) {
   props.PassGetUserInput("Adam", 12)
   return <h1> welcome to react </h1>
  }
  
  ReactDOM.render(<App />, document.getElementById("root"))

As We see we invoke a callback function that we pass from parent component into child component, and from child component we invoke the parent props callback while passing data to parent component

We can pass down function as props into a component for assigning into onClick event handler, this approach is useful if you are planning to use the same component many times and pass one written function to all of them, it will improve the performance instead of writing a function inside each component this will cause a performance issue.

# An Example of Pass a function as props :-

➤ ./src/index.js 


  import React from "react"
  import ReactDOM from "react-dom"
  
  const Button = props => {
   return (
    <div>
     <h1> Click the following button : </h1>
     <button onClick={props.ClickEvent}> Click Me </button> // Good Approach
    </div>
   )
  }
  
  function App() {

  // One function to all components 
   function ClickEvent(e) {
    alert("Clicked")
   }
  
   return (
    <div>
     <Button ClickEvent={ClickEvent} />
     <Button ClickEvent={ClickEvent} />
     <Button ClickEvent={ClickEvent} />
    </div>
   )
  }
  
  ReactDOM.render(<App />, document.getElementById("root"))

The bad approach is defining the function on the component itself, so every component will have it own function event handler it will cause a performance issue rendering each component with it own function.

6 - Using PropTypes Validation

propTypes is Type-checking our props will warn us if we pass a type different from the intended props by the child component. Sending data from one component to another, if a component receives the wrong type of props, it can lead to bugs and unexpected errors in your app.

We were passing props from outside of a component and using them inside that component. But did we have checked what type of values we are getting inside our Component through props? No, we do not. But then also everything worked fine. To make sure that our data that the component receives is valid we use PropTypes validator while exporting a range of validation.

# To start using PropTypes we have to import it first :-

➤ An Example 


  import PropTypes from "prop-types";

  // All following props are optional
  ComponentExample.propTypes = { 
    optionalArray: PropTypes.array,

    optionalElement: PropTypes.element,

  // All following props are required

    RequiredNumber: PropTypes.number.isRequired,
    RequiredObject: PropTypes.object.isRequired,
    RequiredString: PropTypes.string.isRequired,
    RequiredElement: PropTypes.symbol.isRequired,

  // your props is limited to specific values
  ComponentExample.propTypes = { 
    optionalEnum: PropTypes.oneOf(['News', 'Photos']),
  } 
  
  // your props is limited to specific values
  ComponentExample.propTypes = {

      PropTypes.string,

7 - Props Destructuring And Passing Props using Spread operator

The Destructuring was introduced in ES6. It's a JavaScript features that allows us to extract multiple pieces of data from an array or object and assign them to their own variables. let's see how Destructuring works by the following example.

# Example Destructuring Object :-

➤ An Example 


  // Destructuring Object
  let MyObject = { one: 1, two: 2, three: 3 }
  let { one, two, three } = MyObject;

  console.log(one); // output 1
  console.log(two); // output 2
  console.log(three); // output 3

# Example Destructuring Array and spread:-

➤ An Example 


  // Destructuring Array and spread...
  let numbers = [1, 2, 3, 4, 5];
  let [item1, item2, ...items] = numbers;

  console.log(item1); // output 1 
  console.log(item2); // output 2 
  console.log(items); // output [3, 4, 5]

# Example Destructuring Object :-

➤ An Example 


  const person = {
    firstName: "Lindsay",
    lastName: "Criswell",
    city: "NYC",
    code: "002"
  }

  // Destructuring firstName, lastName, city Only 
  const { firstName, lastName, city } = person;

  console.log(firstName); // firstName
  console.log(lastName); // lastName
  console.log(city); // city

# Example Destructuring Object :-

➤ An Example 


  const person = {
    firstName: "Nick",
    lastName: "Anderson",
    age: 35,
    sex: "M"
  }
  
  const { firstName: first, age, city = "Paris", sex = "F" } = person; 
  
  console.log(firstName)  // Undefined -- person.firstName exists BUT the new variable created is named first
  console.log(first)  // "Nick" -- A new variable first is created and is equal to person.firstName
  console.log(age)  // 35 -- A new variable age is created and is equal to person.age 
  console.log(city)  // "Paris" -- A new variable city is created and since person.city is undefined, city is equal to the default value provided "Paris". 
  console.log(sex)  // "M" -- the default is 'F', but it been assigned to ur value 'M'

    loggingIn: false,
   },
  }
  
  function User(props) {
   let { info, auth } = props.data
  
   return (
    <h1>
     {info.name} is {auth.loggedIn ? "login" : "logout"}
    </h1>
   )
  }
  
  ReactDOM.render(<User data={UserSelection} />, document.getElementById("root"))

    <h1> Adam is login </h1>
  </div>

B - Spread Props

You can also spread the props attributes to pass in your component call. If you already have props as an object, and you want to pass it in you component JSX, you can use ... as a spread operator to pass the whole props object.

# An Example of using spread to pass an attribute as object :-

➤ ./src/index.js 


  // Pass props as an attributes 
  function App1() {
    return <Greeting firstName="Ben" lastName="Hector" />;
  }

  // Pass props as object with spread operator 
  function App2() {
    const props = { firstName: "Ben", lastName: "Hector" };
    return <Greeting {...props} />;
  }
