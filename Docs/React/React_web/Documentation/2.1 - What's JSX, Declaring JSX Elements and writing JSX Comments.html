2.1 - What's JSX, Declaring JSX Elements and Writing JSX Comment

1 - What is JSX ?

JSX is a syntax extension to javascript that produce React elements. It is a template script where you will have the power of using HTML and Javascript together. As you know, your web apps are made up of HTML, CSS, and JavaScript It doesn't matter if your web app was written using React or some other library like Angular or jQuery. The end result has to be some combination of HTML, CSS, and JavaScript.

# Why using JSX ?

  • It is faster than normal JavaScript as it performs optimizations while translating to regular JavaScript.
  • It makes it easier for us to create templates.
  • Instead of separating the markup and logic in separated files, React uses components for this purpose.

JSX is a language that allows you to easily mix JavaScript and HTML-like tags to define user interface (UI) elements and their functionality. React uses JSX to handle rendering logic, events, state change in the DOM, and how the data is prepared for display.

# Here is a simple example of a JSX code :-

➤ Example


  const element = <h1> Hello, world! </h1>;

2 - Ways of Declaring JSX

JSX allows us to write HTML elements in JavaScript and place them in the DOM without any createElement() and/or appendChild() methods. You can declare JSX elements or tags in many ways as a Variable, Regular Function, Arrow Function and Anonymous Function

# Declare JSX Elements with variable :-

➤ ./src/index.js


  import React from "react";
  import ReactDOM from "react-dom";

  const element = <h1> Hello, world! </h1>;
  ReactDOM.render(element, document.getElementById("root"));
  <del>ReactDOM.render(<element />, document.getElementById("root"));</del>  // Error

# Add JSX HTML Element inside a Render Method :-

➤ ./src/index.js


  import React from "react";
  import ReactDOM from "react-dom";

  ReactDOM.render( <h1> Hello, world! </h1>, document.getElementById("root"));

# JSX As Regular Function Component :-

➤ ./src/index.js


  import React from "react";
  import ReactDOM from "react-dom";
  
  function App() {
    return <h1> Hello, world! </h1>;
  }
  
  ReactDOM.render(<App />, document.getElementById("root"));
  ReactDOM.render(App, document.getElementById("root"));  // Error

# JSX As Anonymous Function Component :-

➤ ./src/index.js


  import React from "react";
  import ReactDOM from "react-dom";
  
  let App = function() {
    return <h1> Hello, world! </h1>;
  }
  
  ReactDOM.render(<App />, document.getElementById("root"));

# JSX As Arrow Function Component :-

➤ ./src/index.js


  import React from "react";
  import ReactDOM from "react-dom";

    return <h1> Hello, world! </h1>;
  } 

  // OR

  ReactDOM.render(<App />, document.getElementById("root"));

The React Fragment helps in returning multiple elements. It's alternative way to use elements like <div>, <section>, <header> and more. as the following example.

Fragments let you group a list of children without adding extra nodes to the DOM a round them, and serve as a cleaner alternative to using unnecessary div's in your code. We can import React Fragment from React library.

# Fragment render demonstration :-

As you see Fragment let you group a list of elements without adding extra nodes to the DOM because fragments are not rendered to the DOM. So basically we use React.Fragment where we would normally use a wrapper div. We are going to have more examples.

  }

    return (

        <td>ID</td>

    );
  }

  ReactDOM.render(<App />, document.getElementById("root"));


  <div id="root">

      <thead>
        <tr>
          <td>ID</td>

6 - Elements that must be Closed

JSX follows XML rules, and therefore HTML elements must be properly closed. You have to make sure that every element is written properly before compiling you react application, therefore HTML elements without a closing tag must be properly closed with JSX syntax, You must close every empty element that does not have an a closing tag like input, input, br, input, meta and more. let see how to close our JSX tag Elements

# An Example of closing an Html elements without a closing tag using JSX syntax :-

➤ ./src/index.js


  HTML :- <meta charset="UTF-8" >  
  JSX  :- <meta charset="UTF-8" />  
  ----------------------------------------------------------------
  HTML :- <input type="text" name="" id="" >  
  JSX  :- <input type="text" name="" id="" />  
  ----------------------------------------------------------------
  HTML :-  <img alt="my_img" path="/img/pic-1256" alt="" id="image" > 
  JSX  :-  <img alt="my_img" path="/img/pic-1256" alt="" id="image"  />
  ----------------------------------------------------------------
  HTML :-  <i class="fa fa-12" > 
  JSX  :-  <i class="fa fa-12" />
  ----------------------------------------------------------------
  HTML :- <br>
  JSX  :- <br />

8 - React.createElement() Vs JSX

JSX allows us to write HTML elements in JavaScript and place them in the DOM without any createElement() and/or appendChild() methods. JSX Element will be translate into React.createElement() in the background.

# The createElement() method parameters Syntax :-

➤ Syntax


  React.createElement(
    type,
    { attributes || props },
    [...children]
  )

# The createElement() method takes three parameters :-

  • First Parameter - String which specify the HTML tag to be rendered
  • Second Parameter - Pass attributes like class, id, href and also Props. This parameter is optional
  • Third Parameter - The Children of the HTML Element it can be string or an array of elements

The first parameter is the type of element we are creating, it can be another react component. If we are creating an HTML element, we pass in the tag name as a string. The second argument is is an object containing properties, it can be attributes like style, class, id's and href. And also we can pass props that get passed to the component.

Finally the last argument is the children of that component it can be a text type string, or another nested element and components within each other. So let's create our first createElement() component then after that we add attributes and props and nested elements as children.

# An example of using createElement without using jsx :-

➤ An Example


  let myElement = <h1> I Love JSX! </h1>;
  myElement = React.createElement("h1", {}, "I do not use JSX!");

  ReactDOM.render(myElement, document.getElementById("root"));


  <div id="root">
    <h1> I do not use JSX! </h1>
  </div>
