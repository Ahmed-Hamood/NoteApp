2.3 - Using HTML DOM Attributes And Apply CSS Styling in JSX Element

1 - JSX/HTML Attributes

We can write your own custom attributes in addition to regular HTML properties and attributes, before React used to ignore unknown DOM attributes. If you wrote a JSX element with an attribute that React doesn’t recognize, React would just skip it. For example, this:

# Writing Your JSX HTML Elements with unknown attributes. How React 15 & 16 will render unknown attributes in the DOM :-

➤ An Example


  // Your code:
  <div mycustomattribute="something" />

➤ In React 15 DOM Output - Unknown attributes will not be rendered to the DOM


  // React 15 output:
  <div />

➤ In React 16 DOM Output - Unknown attributes will end up to the DOM


  // React 16 output:
  <div mycustomattribute="something" />

Since React components take both custom and related DOM attributes. If your attending to use your own custom attributes that React does not recognize it as a DOM attributes then you have spell it as lowercase so react would know it's a custom attribute. But you can't use React naming reserved keywords as lower case and pass them as a custom attributes to all HTML like className, tabIndex. React will include the attribute in to the DOM but still element won't be changed. for example

# Using React reserved keyword and write them as lowercase, they will be added but elements won't change :-

➤ An Example


  // Yes, it's a class attribute
  <div className="main-header" />

  // Invalid DOM property `classname`. Did you mean `className`?
  <div classname="main-header" />

Q - Should I Keep Data in Custom Attributes ?

NO. We don’t encourage you to keep data in DOM attributes. Even if you have to, better use data-* and aria-* custom attributes that allow you to attach other values onto an HTML element. they're a better approach, but in most cases data should be kept in React component state or external stores.

# Pass data freely thru date and aria attributes :-

➤ An Example


  <div data-foo="42" />
  <button aria-btnlabel="Close" onClick={onClose} />

2 - Canonical React Attribute Naming

React support a custom naming DOM attributes that will translate into javascript attributes and class is a reserved word in JavaScript that we can use inside our HTML element. When JSX is rendered, JSX className attributes are automatically rendered as class attributes. There are a number of attributes that work differently between React and HTML. In React, all DOM properties, attributes and event handlers should be camelCased.

A - readonly attribute - readOnly

The readonly attribute is supported by <input> components of type text. You can use it to prevent anyone to edit the text input, only to view readable input only.

# Apply readOnly attribute with our input element of type text :-

➤ An Example


  HTML :- <input type="password" value="adam" readonly>
  JSX  :- <input type="password" value="adam" readOnly />

B - checked attribute - defaultChecked

The checked attribute is supported by <input> components of type checkbox or radio. You can use it to set whether the component is checked. This is useful for building controlled components. defaultChecked is the uncontrolled equivalent, which sets whether the component is checked when it is first mounted.

# Apply checked attribute with our input element of either type checkbox or radio :-

➤ An Example


  IsMarried: <input type="checkbox" value="IsMarried" defaultChecked />
  IsMarried: <input type="radio" value="IsMarried" defaultChecked />
  ### DOM Output ###
  IsMarried: <input type="checkbox" value="IsMarried" checked>
  IsMarried: <input type="radio" value="IsMarried" checked>

C - class attribute - className

The class attribute has to be written className with an Uppercase format. To specify a CSS class, use the className attribute and insert it within elements like <div> and <a> and others

# Apply className attribute for styling that will be translate into class attribute :-

➤ An Example


  HTML :-  <div id="" class="main-title"> </div>
  JSX  :-  <div id="" className="main-title"> </div>

D - for attribute - htmlFor

The for attribute has to be written htmlFor. Since for is a reserved word in JavaScript, React elements use htmlFor instead.

# Apply htmlFor attribute :-

➤ An Example


  HTML :- <label for="name" class=""> Enter You name </label>
  JSX  :- <label htmlFor="name" className=""> Enter You name </label>

E - style attribute - style

The style attribute takes a reference to an object of camel-cased style properties. It accepts a JavaScript object with camelCased properties rather than a CSS string. This is consistent with the DOM style JavaScript property, is more efficient, and prevents XSS security holes. For example:

# Apply style attribute :-

➤ An Example


  const divStyle = {
    color: 'blue',
    backgroundImage: 'url(' + imgUrl + ')',
  };
  
  function HelloWorldComponent() {
    return (
      <div style={divStyle}>
        Hello World! 
        <p style={{ color: 'red' }}
      </div>;
    ) 
  }

F - selected Attribute - defaultValue

The value attribute is supported by <input>, <select> and <textarea> components. You can use it to set the value of the component. This is useful for building controlled components. defaultValue is the uncontrolled equivalent, which sets the value of the component when it is first mounted.

# Using defaultValue instead of selected in Dropdown List :-

➤ An Example


  <select name="fruits" id="fruit" defaultValue="apple">
    <option value="bananna"> bananna </option>
    <option value="apple"> apple </option>
    <option value="orange"> orange </option>
    <option value="other"> other </option>
  </select>
  ### DOM Output ###
  <select name="fruits" id="fruit">
    <option value="bananna"> bananna </option>
    <option value="apple" selected> apple </option>
    <option value="orange"> orange </option>
    <option value="other"> other </option>
  </select>
