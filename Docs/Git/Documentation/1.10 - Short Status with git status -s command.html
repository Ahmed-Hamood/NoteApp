1.10 - Short Status with git status -s command

We have learned how to get the status of the working directory and the staging area using the git status command. This command output is very comprehensive or thorough. As you see below we will create and modify some files then we are going to run git status command. 

  echo "new text" > file2.txt // create new file called file2.txt
  echo "add new text" >> file1.txt  // modify file called file1.txt
  git status

  On branch master
  Changes not staged for commit:
    (use "git add <file>..." to update what will be committed)
    (use "git restore <file>..." to discard changes in working directory)
          modified:   file1.txt

  Untracked files:
    (use "git add <file>..." to include in what will be committed)
          file2.txt

  no changes added to commit (use "git add" and/or "git commit -a")

As you can see we have a modified file1.txt, and also we have a new file2.txt file un-tracked. The output of our git status command is very comprehensive and very wordy.

We have an alternative way we can supply the git status -s, the short -s dash flag. With this command we will get two column.

  • Left Column: represents another two column "left:right" left means file in the staging area, and right mean file only available in the working directory
  • Right Column: represents the working directory
  git status -s

   M file1.txt  --> ' M' in red means we have a modified file, with changes that are not in the staging area, only available in the working directory
  ?? file2.txt  --> '??' in red means we have untracked file that we need to add it to the stating area

We are going to add both files into the staging area, first lets add file1.txt and then check our short status.

  git add file1.txt
  git status -s

  M  file1.txt  --> 'M ' in green means we have our changed file on the staging area
  ?? file2.txt  --> '??' in red means we have untracked file that we need to add it to the stating area

When we stage a file git takes a snapshot of that file and puts it in the staging area, so if you modify that file after we have to restage the changes. Let modify file1.txt one more time. 

  echo "add new text 2" >> file1.txt
  git status -s

  MM file1.txt  --> 'MM' in green means we have our changed file on the staging area and also we have some additional changes in the working directory that are not added to the staging area
  ?? file2.txt  --> '??' in red means we have untracked file that we need to add it to the stating area

Lets add file1.txt to the staging area one more time, and then run git status -s command, you will see that all the changes that we had in the working directory are now in the staging area. 

  git add file1.txt
  git status -s

  M  file1.txt  --> 'M ' in green means we have our changed file on the staging area
  ?? file2.txt  --> '??' in red means we have untracked file that we need to add it to the stating area

Now, we are done with file1.txt, so lets add untracked file2.txt to the staging area, and then run git status command. 

  git add file2.txt
  git status -s

  M   file1.txt --> 'M ' in green means we have our changed file on the staging area
  A   file2.txt --> 'A ' in green means our file is added to the staging area

As you see git status show that file2.txt is been added to the staging area for the first time. This is how the git status -s command output is simple and brief.
