1.8 - Ignoring Files with .gitignore File

On every project we may have a certain files and directories that we want Git to ignore, we don't to include files that are generated as a result of compiling or while installing addition packages, this is going to increase the size of our repository, and developers may have valuable files that they don't want to share and synchronize with other team members.

We are going create a few files and directories and learn how to configure Git to ignore these files. Lets create a directory called /Log and within it we create a file called info.log, Open your GitBash Terminal and type the following Commands. 

  mkdir logs // Create a new directory called 'logs'
  echo "log info 001" > logs/info.log // Create info.log file inside our logs directory
  git status // check Git Status

  On branch master
  Untracked files:
    (use "git add <file>..." to include in what will be committed)
          logs/
  
  nothing added to commit but untracked files present (use "git add" to track)

When you run git status command, you gonna see that git have un-tracked directory called /log, we don't want to add this folder to the staging area, because we don't Git to track /log folder.

To prevent Git from tracking files and directories, we have to create a special file inside the root of your project called .gitignore, this file has no name it only has an extension. So, lets create this file open your GitBash Terminal again and type the following Commands 

  echo logs/ > .gitignore // create .gitignore file and add a new line
  code .gitignore // open .gitignore in vscode

  logs/

In .gitignore, we have a single entry logs/ which indicates a directory with all it's content that we want to ignore. We can have a list of files and directories or even ignore sub-directory only. Also we can use patterns to ignore files with certain extension. 

  logs/           --> ignore log directory content
  node_modules/   --> ignore node_modules directory content
  users/private/  --> ignore users/private directory, only private content
  main.log        --> ignore main.log file
  process.env     --> ignore process.env file
  *.txt           --> pattern ignore all files with .txt extension 
  *.log           --> pattern ignore all files with .log extension 
  note/*.txt      --> pattern ignore all files with .txt extension inside the note directory 
  note/p1/*.txt   --> pattern ignore all files with .txt extension inside the note/p1/ sub-directory only

After we done, we can save the changes and go back and open the terminal back again

  git status

  On branch master
  Untracked files:
    (use "git add <file>..." to include in what will be committed)
          .gitignore

  nothing added to commit but untracked files present (use "git add" to track)

As you see Git ignored all our files and directories because it's ignoring it instead it says we have a new file called .gitignore. Lets add this file to the staging area and then commit our code with a proper message. 

  git add .gitignore // add .gitignore to the staging area
  git commit -m "add gitignore file" // commit changes to the repository

  [master 8965726] add gitignore file
  1 file changed, 3 insertions(+)
  create mode 100644 .gitignore

Remember, this only works if you haven't already included a file or a directory in your repository, if you include a file in your repository and then later added it to .gitignore Git is not going to ignore that, the file will stay in the staging area

For example If you accidentally include a file in your repository and then later you decided to add it to .gitignore, actually Git is not going to ignore that file because its already been added to the staging area. So, next we will learn how remove files from the staging area that have been added by accident.
