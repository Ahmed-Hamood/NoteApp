1.4 - Committing Changes to Repository with git commit -m "" Command

1. Commit Changes with Simple Message with git commit -m "" Command

We are going to commit our snapchat from the staging area and store it permanently in our Git Repository. We have a snapchat in the staging are ready to be committed in our repository permanently. to confirm that our files in the staging area and ready to be committed we type git status command. 

  git status

  On branch master

  No commits yet
  
  Changes to be committed:
    (use "git rm --cached <file>..." to unstage)
          new file:   file1.txt
          new file:   file2.txt

  git add *

To commit our snapchat in our repository we type git commit -m "initial commit.", we use flag -m for message, then we write our message then we open double quotes, inside it we type a short description that identifies what this snapshot represents, so as this is our first commit then the message will be "initial commit.", you can name it whatever you want. 

  git commit -m "initial commit."

  [master (root-commit) 50cd29e] initial commit.
  2 files changed, 3 insertions(+)
  create mode 100644 file1.txt
  create mode 100644 file2.txt

2. Commit Changes with Detailed Message with git commit Command

sometime you may need to explain your committed snapshot in detailed description or more details, instead of providing a short message that does not explain what is going on. For example if you work on a bug and you want to explain it with your valuable solution while committing you code.

With situations like this we drop git commit -m "bugs fix", and we only just type git commit command. After you press enter our default editor will open a file called "COMMIT_EDITMSG" stored in .git sub-directory, here we can edit file commit. 

  git commit 
  // hint: Waiting for your editor to close the file... 

 >> .git/COMMIT_EDITMSG

  # Please enter the commit message for your changes. Lines starting
  # with '#' will be ignored, and an empty message aborts the commit.
  #
  # On branch master
  # Changes to be committed:
  #	modified:   file2.txt
  #

Texts After # sign are considered as a comment, we can write our description any where. We can specify title them below write a long description as you see below. 

  Fix Bugs:

  Our bugs occur because the string value ......
  # Please enter the commit message for your changes. Lines starting
  # with '#' will be ignored, and an empty message aborts the commit.
  #
  # On branch master
  # Changes to be committed:
  #	modified:   file2.txt
  #

  // you can remove all comment and add new lines

  Fix Bugs:

  Our bugs occur because the string value ......

Then, we save the changes and then we close COMMIT_EDITMSG file window, our snapshot will committed and git will return a message statists about what was changed. As you see below 2 insertion or two lines were inserted 

  git commit 

  [master 78822ce] Fix Bugs
   2 files changed, 2 insertions(+)

After you commit snapshot changes into your repository, our project terminal will tur green, which mean it doesn't have any changes so what we have in pur working directory is exactly the same content we have in the staging area which is exactly te same content we have in the last commit.
