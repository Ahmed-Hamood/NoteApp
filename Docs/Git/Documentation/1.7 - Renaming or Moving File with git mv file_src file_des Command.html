1.7 - Renaming or Moving File with git mv file_src file_des Command

In Git renaming or moving files is a two-step operation, first we have to modify our working directory and then we have to stage two type of changes an addition and a deletion Git provides a special command that does both operation in one step, the git mv file_src file_des command. Lets check both operations first before we learn about our special command.

We have a single file in our working directory, and that is file1.txt we are going to rename it to main.js by using the mv command in unix to rename file1.txt. First let's view our project folder content. 

  ls

  file1.txt

With mv command we can Rename or Remove files and directories.

  mv file1.txt main.js

Let's run git status and see our changes, will see that we have two changes both are un-stages because they are indicated by red. We a have file1.txt with delete operation, and under untracked files we have a new file called main.js as we know Git doesn't automatically track all your new files 

  git status

  On branch master
  Changes not staged for commit:
    (use "git add/rm <file>..." to update what will be committed)
    (use "git restore <file>..." to discard changes in working directory)
          deleted:    file1.txt
  
  Untracked files:
    (use "git add <file>..." to include in what will be committed)
          main.js
  
  no changes added to commit (use "git add" and/or "git commit -a")

Every time you have a new file in your project you have to add it to the staging area, so Git starts tracking it. So we need to use the git add command, to stage both these changes. 

  git add file1.txt // Stage file1.txt deletion
  git add main.js // Stage main.js addition
  git status

  On branch master
  Changes to be committed:
    (use "git restore --staged <file>..." to unstage)
          renamed:    file1.txt -> main.js

Now, Git recognized that ewe renamed file1.txt to main.js, as you see renaming is indicated by green which means this is in the staging area, and ready to commit it to the repository. 

  git commit -m "file Rename from file1.txt to main.js"

  [master f9adf6f] file modify from file1.txt to main.js
  1 file changed, 0 insertions(+), 0 deletions(-)
  rename file1.txt => main.js (100%)

Let's do renaming again, but with our special git mv command to rename or move our file in one step, instead of using the standard Linus/Unix mv command, We are going to use the git mv file_src file_des command. 

  git mv main.js file1.txt
  git status

  $ git status
  On branch master
  Changes to be committed:
    (use "git restore --staged <file>..." to unstage)
          renamed:    main.js -> file1.txt

We renamed main.js file into file1.txt without the need to commit multiple changes. Our git mv command the changes are applied to both the working directory amd the staging area. Now let's commit the changes. 

  git commit -m "Refactor code"

  [master 3dc0bb3] Refactor code
  1 file changed, 0 insertions(+), 0 deletions(-)
  rename main.js => file1.txt (100%)

If you look at the statistics, one file has changed and we have zero insertions because we haven't added any new lines to any of our files, and also have a zero deletions because we haven't remove any lines from any files
