Getting Start

1. What is Git ?

Git is the most popular version control system, which records the changes made to our code over time in a special database called repository. Our project will have a history that allow us to see how made what changes when and why.

If someone screw up something in our code, we can simply and easily revert our project back to an earlier state. So without a version control system we would constantly store copies of the entire project in various folders, this will be very slow and doesn't scale where multiple developers can't work in the same folder folder and you can review the latest code that our developer come up with in the same project with different files to merge both changes.

With Version Control System like Git we can track our project history where a group of developers can work together in the same project. Our Version Control System has two types or categories centralized and distributed.

# We use git because :

  • Free of cost
  • Open source
  • Super fast
  • Scalable and maintainable
  • Cheap Branching/Merging

- Types of Version Control System

Centralized System: Where multi-members connect to a central server to get the latest copy of the code, and to share their changes with others. The Sub-version and microsoft team foundation server are examples of centralized version control systems. The problem with centralized architecture is the single point of failure if the server goes offline then we can't collaborate or save latest updated code/snapshots of our project, so we need to wait until the server comes back online

Distributed System: Here every one has a copy of the project with its history on their machine, so we can save snapshots of our project locally on our machine, if the central server is offline we can synchronize our work directly with others. Distributed System like Git and Mercurial are the most popular Version Control System

- How to use Git ?

We can use get in terminal or command prompt window to execute git commands, most developer uses git command thru the command line. Also we have a build-in extensions git features within the code editor or IDE, VSCode supports git with a source control panel extensions which gives us the essential git features, also there are also extensions available for an additional git features for vscode like GitLens extension.

We also have GUI Git client tools for different platforms, the most popular tools is GitKraken Git GUI and SourceTree, they present our project history as a timeline, and we can track changes easily and manage each version graphically.

# Why we are going to git command thru the command line instead of GUI or extension IDE :-

  • GUI Tools have limitation
  • GUI Tools are not always available like when using a remote server

3. Configure Git

Before we start initializing git on our project, first we need to specify few configuration settings like Name, Email, Default Editor and Line Ending. We can configure these different setting at three different levels.

- Configuration System Level :-

  • System Level- This is a very top level, Which apply configuration to all users of the current machine
  • Global Level- This level apply to all repositories/projects of the current user
  • Local Level- This level only applied on the current repository or the repository in the current project folder

- Configure System Setting Level with name

With Local level, we can have a different settings for different repositories or different projects. We will apply our setting at global level which apply to all repository of the current user only. Open you terminal and configure your level by specifying --global which specify the level that will define our setting on. After that we should specify the settings user.name setting. 

  git config --global user.name "Adam William"

- Configure System Setting Level with email

The reason we adding double quote is because we have a space in our user.name value. once again we repeat our configuration global level and specify our email user with user.email, but this time we don't need to wrap our value string between double quotes because it's an email so we don't have a space in an email id. 

  git config --global user.email adam123@gmail.com

- Set the default Editor with Git

Let's set the default editor with git we set or configure the core.editor setting and assign "code --wait", the wait flag tells the terminal window to wait until we close a new vscode instance 

  git config --global core.editor "code --wait"

- Open VSCode Editor or Default Editor

Next, we are going to specify our default editor, by default in MACOS is going to use vim. In windows we are going to use VSCode editor, so download it by visiting https://code.visualstudio.com/download, now access your project folder by specifying it's path, and type the following to open it with vscode. 

  code // just open vscode
  code . // open vscode on the current path using "dot"

- Configure End of Lines on Window and MacOS/Linux

we going to configure how git should handle end of lines, which is an important setting that a lot of people miss. On windows end of lines are marked with two special characters as following.

# End of lines :

  • The "\r" CR or carriage return, it moves the cursor to the beginning of the line.
  • The "\n" LF or line feed, it moves moves the cursor to the next line

On MacOS/Linux Systems end of line is indicated with line feeds, so that means if we don't handle end of line properly then we're going to run into some weird issues down the road to prevent this we have to configure a property called code.autocrlf which is a short for carriage return / line feed.

  • Window indicate end of line with "\n" and "\r"
  • MacOS/Linus indicate end of line with "\n" Only

Let's say two developer working with the same repository one developer uses a window and other developer uses a MacOS. In windows the end of lines are marked with carriage return and line feed. If window user wants to check his code into the repository Git to remove the carriage return from end of lines, similarly checking out the code from the repository Git should update end of line and add the carriage return character, to achieve this behavior we should set the core.autocrlf property into a true value.

In other hand MacOS user doesn't want the carriage return character, so Git should not touch end of lines, if MacOS developer get carriage return character where Window is using, then Git should remove it when storing the code in the repository, we should set the core.autocrlf property into a input value 

  git config --global core.autocrlf true // In windows
  git config --global core.autocrlf input // In MacOS/Linux

- Open Git File Global Settings in default editor

All configurations are stored in a text file, we will vscode editor to edit that file, to open this file in our default editor we use "-e" flag. 

  git config --global -e

  C://Users//username//.gitconfig

  [core]
  editor = code --wait
  autocrlf = true

  [user]
  name = Username
  email = email@gmail.com

The .gitconfig file has different sections, like the user section with two settings name and email, and also the core section with two settings editor and autocrlf settings. 

  git config --global --list

  core.editor=code --wait
  core.autocrlf=true
  user.name=Username
  user.email=your_email@gmail.com

- Getting Help with Git commands

To get help about Git commands, lets get help with config command, we simply type git config --help, this will give you an information about the config command. You can use -h this will give us a short summary and its options. 

  git config --help
  git config -h

The command above, will print information on the terminal, or will direct you to Git website.

4. Initializing Git Repository

Lets say we decided to use Git with our project, first thing we do is create a directory of our project any where in your machine. To create a directory using terminal command type the following command with GitBash emulator. 

  mkdir MyProject // Create Directory Project called "MyProject"
  cd MyProject // Enter our project

Now within the project directory where all our file will be added, to start using Git repository in out project folder we have to initialize a new empty repository. So we type git init command to initialize an empty repository 

  git init

  Initialized empty Git repository in C:/Users/username/Desktop/MyProject/.git/

A hidden folder is created inside our project folder called .git which is a su-directory, this folder is hidden because you are not suppose to touch it. Inside .git directory/repository here where git stores information about our project history. Don't remove the .git directory or you will lose your project history
